Thinking in React는 리액트 컴포넌트를 설계하는 방법에 대해 기술한 문서입니다. 리액트를 처음 접하는 사람이라면 컴포넌트를 어떻게 설계해야 할지 막막할 수 있는데, 이 문서를 통해 순차적으로 컴포넌트를 설계하는 방법을 배워볼 수 있습니다.

## 1. UI를 컴포넌트 계층으로 쪼개기

자 우선 키보드에서 손 떼고 펜과 종이를 준비하자. 잘 다룰줄 아는 시각화 도구가 있다먼 그걸 활용해도 된다. 피그잼이나 키노트, 또는 코드 에디터의 스타일링 기능을 활용해도 된다.

리액트 컴포넌트를 만들기 위해 첫번째로 해야할 것은 UI를 컴포넌트 계층으로 쪼개는 것이다. 이 과정을 통해 각 컴포넌트의 책임과 범위를 명확히 할 수 있다.

### 어떤 기준으로 올릴까, 내릴까?

컴포넌트를 계층화하는 기준은 다음과 같습니다:

1. 단일 책임 원칙: 각 컴포넌트는 하나의 책임만 가져야 합니다. 만약 컴포넌트가 여러 가지 일을 하고 있다면, 더 작은 컴포넌트로 분리해야 합니다.

2. 재사용성: 여러 곳에서 사용될 수 있는 UI 요소는 별도의 컴포넌트로 만듭니다. 이는 코드 중복을 줄이고 유지보수를 용이하게 합니다.

3. UI 구조: UI의 시각적 구조를 따라 컴포넌트를 나눕니다. 예를 들어, 헤더, 사이드바, 메인 콘텐츠 등으로 나눌 수 있습니다.

4. 데이터 흐름: 데이터를 주고받는 단위로 컴포넌트를 나눕니다. 이는 상태 관리를 더 쉽게 만들어줍니다.

5. 복잡성 관리: 너무 복잡해지는 컴포넌트는 더 작은 단위로 나누어 관리합니다. 이는 코드의 가독성과 유지보수성을 높입니다.

이러한 기준을 고려하여 컴포넌트를 계층화하면, 더 체계적이고 관리하기 쉬운 리액트 애플리케이션을 만들 수 있습니다.

### props 와 state

정적인 버전의 리액트 앱에는 state가 없다. state는 상호작용이 추가된 버전의 리액트 앱에서만 쓰는것이다. 즉, 2번 단계까지는 useState 훅을 사용하지 않는다.

규모가 큰 react 어플리케이션을 만들다 보면 props drilling이라는 문제에 자주 직면하기에 props 자체를 사용하지 않는 경우가 많다.
